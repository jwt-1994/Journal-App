## Context

当前 P3A 已完成拼贴编辑器核心功能，但整体 UI 结构存在以下问题：
- 导航菜单有 7 个条目（素材库、上传、分类管理、背景库、报表看板、拼贴、设置），过多且功能分散
- 素材库显示原图+抠图混合，用户实际只需要抠图后的透明图案
- 背景库与素材库分离，但"背景"本质也是素材的一种
- 拼贴画布创建时手动输入宽高对普通用户不友好
- 拼贴方案入口叫"拼贴"，不够直观

## Goals / Non-Goals

**Goals:**
- 精简菜单到 5 个核心入口：素材库、上传、手账本、拼贴、设置
- 素材库按分类标签页展示，只显示抠图后的图案，缩略图+大图预览
- 背景库合并为素材库的"背景"分类标签
- 拼贴画布使用预设尺寸类型（M5、A6、TN 等）
- 上传时支持手动输入素材名称

**Non-Goals:**
- 不改变后端 API 整体结构，仅做参数扩展
- 不涉及 AI 增强、虚拟滚动等 P3B 内容
- 不改变拼贴编辑器核心逻辑（画布渲染、工具系统）

## Decisions

### 1. 导航菜单合并方案

**决定：** 使用精简后的导航结构

```
素材库 → 分类标签页预览（含背景分类）
上传   → 上传素材（可输入名称）
手账本 → 已保存的拼贴方案列表
拼贴   → 拼贴编辑器（新建画布 → 编辑 → 保存）
设置   → 分类管理 + 通用设置
```

**替代方案：** 素材库+上传合并为一个页面 → 不采用，上传和浏览是两种独立操作流

### 2. 素材库 v2 架构

**决定：** 使用 Ant Design Tabs 组件，每个分类一个 Tab，再加一个"全部"Tab 和"背景"Tab

```
┌──────────────────────────────────────────────┐
│ 搜索框 + 排序下拉                            │
├──────────────────────────────────────────────┤
│ [全部] [分类A] [分类B] [背景] [分类C] ...    │
├──────────────────────────────────────────────┤
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐                        │
│  │  │ │  │ │  │ │  │  ← 抠图后小图(120px)   │
│  └──┘ └──┘ └──┘ └──┘                        │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐                        │
│  │  │ │  │ │  │ │  │                        │
│  └──┘ └──┘ └──┘ └──┘                        │
├──────────────────────────────────────────────┤
│  分页器                                      │
└──────────────────────────────────────────────┘
```

- 只显示 `has_removed_bg === 'done'` 的素材（抠图成功）
- 点击缩略图弹出 Modal 大图预览（最大 600px）
- 搜索和排序保留现有逻辑，在 Tab 切换时独立生效
- 背景 Tab 显示背景库数据，同样缩略图展示

### 3. 背景库合并

**决定：** 背景库的后端 API 和模型不变，前端通过"背景"Tab 展示，作为素材库的一个特殊分类

- 素材库 `GET /api/materials` 已有 `has_removed_bg=done` 默认筛选
- 背景 Tab 额外请求 `GET /api/backgrounds` 独立渲染
- 拼贴编辑器中背景选择仍从 backgrounds API 获取

### 4. 画布预设尺寸

**决定：** 使用预设类型选择器替代手动输入，预设映射表：

| 类型 | 宽度 | 高度 | 说明 |
|------|------|------|------|
| M5   | 1050 | 1485 | 系统手账 |
| A6   | 1050 | 1485 | 小本 |
| A5   | 1485 | 2100 | 大本 |
| TN 标准 | 990  | 2100 | Traveler's Notebook |
| TN 护照 | 890  | 1240 | TN 护照尺寸 |
| B6   | 1250 | 1760 | 中型本 |
| 方形 | 1200 | 1200 | 正方形 |
| 自定义 | 用户输入 | 用户输入 | 自由尺寸 |

**替代方案：** 完全去掉自定义 → 不采用，保留灵活性

### 5. 上传素材名称

**决定：** 上传表单增加可选"素材名称"输入框，传递到后端 `POST /api/materials` 的 `name` 参数

- 后端：`name` 字段存入 `original_name` 列（如果为空则使用原文件名）
- 前端：上传成功后显示已输入的名称

## Risks / Trade-offs

- **风险：** 素材库只显示抠图后素材，用户可能无法看到未处理或处理中的素材 → **缓解：** 增加"全部"Tab 显示所有素材（用图标标记抠图状态）
- **风险：** 背景库数据与素材库数据结构不同，合并展示可能混乱 → **缓解：** 背景 Tab 使用独立数据源，视觉上与素材 Tab 一致但内部逻辑分离
- **风险：** 移除报表看板菜单后用户无法查看统计 → **缓解：** 后端 API 保留，后续可在设置页增加简要统计入口
