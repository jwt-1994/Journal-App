## Context

P0 阶段已实现素材上传、分类、浏览、报表功能。P1 在现有架构上增加抠图能力。rembg 是 MIT 许可的开源抠图工具，基于 U²-Net 深度学习模型，对物品/贴纸类素材效果良好，完全离线运行无需 API 调用。

## Goals / Non-Goals

**Goals:**
- 集成 rembg 实现自动抠图
- 上传后异步触发抠图，不阻塞用户操作
- 支持抠图状态追踪（处理中/成功/失败）
- 支持手动重试失败的抠图
- 前端展示抠图状态和原图/抠图对比
- 抠图失败时保留原图，素材仍可正常使用

**Non-Goals:**
- 不支持视频抠图
- 不支持批量抠图（单张逐个处理即可）
- 不替换抠图引擎（当前仅 rembg）

## Decisions

### 1. 抠图引擎：rembg
- **理由**: MIT 开源免费，离线运行，U²-Net 模型对手账素材效果好
- **安装方式**: `pip install rembg`，首次运行自动下载模型（~176MB）
- **替代方案**: 在线 API（要钱/网络依赖），SAM（太重，需 GPU）

### 2. 抠图时机：上传后异步执行
- **理由**: 抠图耗时 2-10 秒/张，同步执行会阻塞上传体验
- **实现**: 使用 Python `threading.Thread` 启动后台线程执行抠图
- **替代方案**: Celery（太重），asyncio（CPU 密集型不适合）

### 3. 抠图状态模型
- 在 Material 表增加字段而非新建表，简化查询
- `has_removed_bg`: 枚举（none/processing/done/failed）
- `removed_bg_path`: 抠图成功后的文件路径

### 4. 文件存储
- 抠图结果存 `data/processed/{original_filename}.png`（强制 PNG 以保留透明通道）
- 原图保持不动，抠图结果作为附加文件

### 5. 前端展示
- 素材库卡片上增加小标签表示抠图状态
- 素材详情模态框中增加"原图/抠图"切换按钮
- 上传页面增加"自动抠图"开关（默认开启）

## Risks / Trade-offs

- **[首次运行慢]**: rembg 首次运行需下载模型 ~176MB → 在 Dockerfile 中预下载，或首次启动时提示
- **[模型体积]**: 后端 Docker 镜像体积增大 ~200MB → 可接受，桌面应用无网络传输压力
- **[抠图质量]**: 对低对比度素材效果可能不佳 → 保留原图，用户可手动判断
- **[CPU 占用]**: 抠图时 CPU 占用较高 → 单张串行处理，避免并行
